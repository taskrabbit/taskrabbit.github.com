Accept an offer on a Task

TaskRabbit allows the acceptance of an offer on an authorized users posted task.

Setup

  • There is a user who is authenticated to your client application with id "jhfe92kjfklep902mrdfg3jdsuo32b1os"
  • The client application secret is "fei3on2o1in321XOIENOIn2io13zoin21x"
  • The user has an authorization token of "rRou72pewt102y926jfhwow438238poQR31d"
  • The city of the user is Boston
  • The user has an opened task "My Task" with id 1452
  • There is an offer on the task with id 213
  • The city of the runner is Boston

POST /api/v1/tasks/1452/offers/213/accept

  • Authorization: OAuth rRou72pewt102y926jfhwow438238poQR31d

Simplified Response

  • Status Code: 200
  • Content-Type: application/json
{
  "id": 213,
  "charge_price": 15,
  "state": "accepted",
  "task": {
    "name": "My Task",
    "named_price": 10,
    "runner": {
      "id": 3576,
      "city": {
        "id": 1,
        "name": "Boston"
      }
    },
    "state": "assigned",
    "city": {
      "id": 1,
      "name": "Boston"
    }
  }
}

Error Cases

The server will respond with different status codes when there is a problem accepting an offer.

Incorrect Authorization

POST /api/v1/tasks/1452/offers/213/accept

  • Authorization: OAuth XXXXXX
  • X-Client-Application: fei3on2o1in321XOIENOIn2io13zoin21x

401 Unauthorized

  • Content-Type: application/json
{
  "error":"There must be an authenticated user for this action"
}
